An Interactive Biomedical Search Engine

About Setation:

Setation works by separately compiling MEDLINE search results for each query input term. Resulting abstracts for each search are separated into words, phrases up to four words, and MeSH (medical subject headings) terms. These items are counted and stored in a file for each input. All files are then combined to produce the final results. The resulting list is sorted by the largest network of input terms that contain each item and then by total frequency. Setation uses pattern recognition, statistical methods, and database comparison algorithms to find the most common implicit keywords, potentially associated genes and diseases, and MeSH terms for the set. Data network visualizations using Cytoscape Web enable the users to comprehend the complex interactions among the results. Medline literature is accessed using the Entrez API and BioPython.




Table 1. Input recognition results for BRCA1 and BRCA2

Each input is analyzed for type including gene, disease, and treatment. Gene information such as other aliases, description, and organism are provided. Top results for associated diseases and genes are returned for each input. Analysis of each input is important to complete before full list analysis. If the input is not found in a categorical database, learning algorithms attempt to place the input according to data signature.
